# Site-Migrations-Kenntnisse {#site-migration}

Durch die Site-Migrationsfähigkeit werden Websites mithilfe von KI und Adobe Forward Deployed Engineers (FDEs) in Edge Delivery Services integriert. Die Kenntnisse automatisieren die Migration von Inhalten und Stilen und bereiten eine autorenfertige Edge Delivery Services-Implementierung vor, die Sie über die Code-AEM-Bearbeitung weiter anpassen können.

## Überblick {#overview}

Durch die Site-Migrationsfertigkeit wird eine vorhandene Site oder ein vorhandener Entwurf in ein Edge Delivery Services-Projekt umgewandelt. Eine Migration, bei der die Kenntnisse verwendet werden, umfasst die folgenden Komponenten.

* **Quellen**: Eine bestehende Website (alle CMS, einschließlich AEM Publish), ein Design-System oder Design-Pseudodateien (z. B. Figma)
* **Ausgaben**: Ein produktionsfähiges Edge Delivery Services-Repository mit angewendeten Blöcken und Stilen
* **Ziele**: Alle Edge Delivery Services-Plattformen und Authoring-Methoden werden als Ziele unterstützt
* **Betriebsmodell** Wird von vornherein bereitgestellten Adobe-Technikern bereitgestellt, die den Agenten betreiben und die Migration leiten, um Qualität und Governance sicherzustellen

## Funktionen {#capabilities}

Die Site-Migrationsfertigkeit führt die folgenden allgemeinen Aufgaben aus:

1. **Inhaltsmigration** - Konvertiert Seiten und Assets in Edge Delivery Services-Inhalte, die mithilfe von Standardinhalten strukturiert sind, und Blöcke, die an der ausgewählten Blockbibliothek ausgerichtet sind.

2. **Stilprogramm** - Sie erstellt Stile für Edge Delivery Services auf eine der folgenden Arten:

   * **Migration im Istzustand** von der aktuellen Site
   * **Redesign** unter Verwendung der bereitgestellten Nachahmungen (z. B. Figma)
   * **Wiederverwenden vorhandener Blockdesigns** aus einer vorhandenen Edge Delivery Services-Implementierung

Typische Ergebnisse sind:

* Edge Delivery Services-Repository (Blöcke, Stile, Konfiguration)
* Migrierte Inhalte und Assets
* Authoring-Konfiguration (Dokumenterstellung oder universeller Editor)
* Edge Delivery Services-Umgebungen (Vorschau und Live-Schaltung) mit der bereitgestellten migrierten Site

Das Ergebnis ist eine produktionsbereite Website, die für Leistung (Core Web Vitals/Lighthouse), Barrierefreiheit und responsive Layouts über Breakpoints hinweg optimiert und über die Dokumenterstellung oder den universellen Editor vollständig bearbeitbar ist.

## Einschränkungen {#limitations}

Die Fähigkeit zur Site-Migration konzentriert sich auf Inhalt und Stil. Die folgenden Anwendungsfälle erfordern zusätzlichen Implementierungsaufwand zusätzlich zu den Fähigkeiten zur Site-Migration:

* Benutzerdefinierte Integrationen (z. B. Commerce, CRM, Such-Connectoren)
* Dynamische Inhalte oder Rendering (z. B. asynchron geladene Inhalte, Einzelseitenanwendungen oder maßgeschneiderte JavaScript-Frameworks)
* Neue Funktionsentwicklung, die nicht mit dem migrierten Erlebnis zusammenhängt

Wenn Sie diese Funktionen benötigen, können Adobe Consulting Services oder Ihr Implementierungspartner das bereitgestellte Edge Delivery-Projekt erweitern.

## Interaktionsmodell {#engagement}

Die Site-Migrationsfertigkeit ist derzeit über ein Projekt verfügbar, das von durch Adobe Forward Deployed Engineers verwaltet wird. Jedes Projekt beginnt mit einer Umfangsübung, die auf der Komplexität der Quelle, den Design-Zielen, der Skalierung der Inhalte und den Governance-Anforderungen basiert.

So untersuchen Sie eine Migration:

* Wenden Sie sich an Ihren Adobe-Support-Mitarbeiter oder Ihr Account-Team, um die Berechnung und Planung einzuleiten.
* Adobe wird die Eignung bestätigen, das Projekt schätzen und einen Migrationsplan vorschlagen.
