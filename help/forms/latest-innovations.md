# Die wichtigsten Innovationen in Adobe Experience Manager Forms {#latest-innovations}

Zu den wichtigsten Innovationen in Adobe Experience Manager Forms gehören:

| Innovation | Details |
|---|---|
| Edge-Bereitstellung | Mit der Edge-Bereitstellung können Mitarbeitende in Ihrem Unternehmen Formulare in ihren bevorzugten Tools wie Microsoft Excel und Google Sheets erstellen, ohne dass umfangreiche Schulungen zu neuen UI-Workflows erforderlich sind. <br/> <br/> Darüber hinaus wurde die Edge-Bereitstellung entwickelt, um Formulare mit unübertroffener Geschwindigkeit zu laden und so einen nahezu perfekten Google Lighthouse-Wert auf Live-Seiten zu erzielen, wodurch eine optimale Leistung und ein optimales Anwendererlebnis gewährleistet ist.<br/> <br/> ![Tools für die Kombinierbarkeit der nächsten Generation zur Erstellung von Formularen](/help/forms/assets/web-vitals.jpeg) Darüber hinaus war die Bereitstellung von Formularen dank der einfachen Tools der Edge-Bereitstellung noch nie einfacher oder schneller. Mit minimaler Entwicklungszeit können Sie den Weg bis zur Live-Schaltung einfach optimieren und Ihren Kundinnen und Kunden außergewöhnliche Formulare bieten. |
| Adobe Workfront Fusion Connector | Der [AEM Forms Adobe Workfront Fusion Connector](/help/forms/submit-adaptive-form-to-workfront-fusion.md) ermöglicht die nahtlose Integration zwischen Adobe Experience Manager (AEM) Forms und Adobe Workfront Fusion. Adobe Workfront dient als Work-Management-Anwendung, die den gesamten Arbeitszyklus zentralisiert, während Workfront Fusion als Integrationsplattform fungiert, die die Verbindung zwischen Workfront und verschiedenen Geschäftsanwendungen vereinfacht. <br/> ![Adobe Workfront](/help/forms/assets/adobe-workfront.png) <br/> Mit dem Adobe Workfront Fusion Connector können Sie Workflows entwickeln, die automatisch bei der Übermittlung eines adaptiven Formulars ausgelöst werden. Stellen Sie sich beispielsweise ein Szenario vor, bei dem ein Workflow initiiert wird, um einer bestimmten Person die Aufgabe zuzuweisen, die übermittelten Daten zu überprüfen, sodass ein Antrag auf Grundlage der im adaptiven Formular erfassten Informationen genehmigt oder abgelehnt werden kann. Diese optimierte Integration verbessert die Effizienz und führt die Automatisierung Ihrer Workflow-Prozesse auf eine neue Stufe. |
| Headless Adaptive Forms | Erstellen und verwalten Sie [Headless Adaptive Forms](https://experienceleague.adobe.com/docs/experience-manager-headless-adaptive-forms/using/overview.html?lang=de) innerhalb der Adobe Experience Manager-Plattform. Ermöglichen Sie es Entwickelnden, interaktive Formulare zu erstellen, zu veröffentlichen und zu verwalten, bei denen Zugriffe und Interaktionen statt über eine herkömmliche grafische Benutzeroberfläche über APIs erfolgen. <br/> <br/> Diese Formulare erfordern zur Übermittlung keine herkömmliche HTML-Formularoberfläche. Mit anderen Worten: Sie ermöglichen es Ihnen, Formulardaten programmgesteuert über eine API oder einen Backend-Code zu senden, ohne dass sichtbare Formularelemente am Frontend erforderlich sind. <br/> <!-- ![Headless adaptive form working](https://experienceleagueadobe.com/docs/experience-manager-headless-adaptive-forms/assets/how-headless-adaprive-forms-work.png?) --><br/> Headless-Formulare sind in verschiedenen Szenarien nützlich, z. B. beim Erstellen von Einzelseitenanwendungen, progressiven Web-Apps oder Mobile Apps, bei denen eine herkömmliche HTML-Formularoberfläche vielleicht nicht erforderlich oder unpraktisch ist. Da Entwickelnde Formulardaten direkt über APIs oder Backend-Code senden können, verbessern Headless-Formulare Workflows und die allgemeine Leistung von Web-Anwendungen. |
| Kernkomponenten | Die [Kernkomponenten von adaptiven Formularen](https://experienceleague.adobe.com/docs/experience-manager-core-components/using/adaptive-forms/introduction.html?lang=de#features) bestehen aus 24 Open-Source- und BEM-kompatiblen Komponenten, die auf der Grundlage der Adobe Experience Manager WCM-Kernkomponenten erstellt wurden. Sie wurden speziell für die Erstellung von adaptiven Formularen entwickelt. Hierbei handelt es sich um Formulare, die sich an Gerät, Browser und Bildschirmgröße des Benutzenden anpassen. <br/> <br/> Diese Komponenten können verwendet werden, um außergewöhnliche Datenerfassungs- und Registrierungmöglichkeiten zu schaffen, indem sie eine Fülle von Formularfeldoptionen bereitstellen, darunter Textfelder, Kontrollkästchen und Dropdown-Menüs. Sie beinhalten auch Funktionen wie Validierung, konditionelle Logik und responsives Design, mit denen sich Formulare erstellen lassen, die benutzerfreundlich und einfach zu verwenden sind. <br/> ![Funktionen von Kernkomponenten](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/assets/sample-core-components-based-adaptive-form.png?)<br/> Da es sich bei diesen Komponenten außerdem um Open-Source-Komponenten handelt, können Entwickelnde die Komponenten einfach anpassen und erweitern, um sie an die spezifischen Anforderungen ihrer Organisation anzupassen. Darüber hinaus basieren diese Komponenten auf der BEM-Methode, die sicherstellt, dass sie skalierbar und wartbar sind. |
| Microsoft PowerAutomate Connector | Mit dem AEM Forms Power Automate Connector können Sie die Integration von Adobe Experience Manager (AEM) Forms in Microsoft Power Automate (zuvor als Microsoft Flow bekannt) vornehmen. Power Automate ist ein Cloud-basierter Dienst, mit dem Sie automatisierte Workflows zwischen verschiedenen Anwendungen und Diensten erstellen können. <br/> <br/> Mit dem AEM Forms Power Automate Connector können Sie Workflows erstellen, die automatisch bei der Übermittlung eines adaptiven Formulars ausgelöst werden. Sie können beispielsweise einen Workflow einrichten, der automatisch eine E-Mail-Benachrichtigung an eine bestimmte Person sendet, wenn jemand ein Formular sendet. Ebenso können Sie eine Aufgabe im Microsoft Planer erstellen, wenn ein Formular ausgefüllt wird. <br/> ![Symbol von Microsoft Power Automate Connector](https://powerusers.microsoft.com/t5/image/serverpage/image-id/182924i17C4BEA1C045D731/image-size/large/is-moderation-mode/true?v=1.0&amp;px=999)<br/>Der AEM Forms Power Automate Connector ist ein leistungsstarkes Tool, mit dem Sie Ihre adaptiven Formulare automatisieren und mit anderen Anwendungen und Diensten integrieren können, die mit Microsoft Power Automate verbunden sind, sodass Sie mit einer größeren Palette von Tools arbeiten können. Sie können Workflows erstellen, die auf Ihre spezifischen Anforderungen zugeschnitten sind, sowie benutzerdefinierte Aktionen, Bedingungen und Trigger hinzufügen. Zusätzlich bietet Power Automate detaillierte Analysen und Berichte, mit denen Sie Ihre Workflows langfristig überwachen und optimieren können. |
| Microsoft Storage Connectoren | Mit AEM Forms Microsoft Storage Connectoren für <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html?lang=de#submit-to-sharedrive">OneDrive</a>, <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html?lang=de#submit-to-sharedrive"> SharePoint, </a> und <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html?lang=de#submit-to-azure-blob-storage"> Azure Blob-Speicher </a> können Sie Adobe Experience Manager (AEM) Forms mit Microsoft OneDrive und SharePoint integrieren. Mit diesem Connector können Sie Datendateien und Anhänge direkt von adaptiven Fomularen in OneDrive und SharePoint hochladen. <br/> ![Symbol von Microsoft OneDrive und SharePoint](/help/forms/assets/onedrive-and-sharepoint.jpg) <br/>OneDrive und SharePoint können in andere Geschäftsanwendungen wie CRM-Systeme, Buchhaltungs-Software und Projekt-Management-Tools integriert werden. Damit können Sie Ihre Geschäftsprozesse optimieren, die manuelle Dateneingabe reduzieren und die Gesamteffizienz verbessern. |
| Benutzeroberfläche des Assistenten | Die Benutzeroberfläche des Assistenten von Adaptive Forms ist ein leistungsstarkes Tool zum schnellen und einfachen Erstellen adaptiver Formulare. Die benutzerfreundliche Oberfläche und die Anpassungsmöglichkeiten machen sie für alle Benutzenden, unabhängig von ihrem technischen Fachwissen, zugänglich. <br/> <br/> Die Benutzeroberfläche des Assistenten vereinfacht das Erstellen adaptiver Formulare, indem sie die Benutzenden Schritt für Schritt durch den Prozess der Formularerstellung führt. Die Benutzeroberfläche des Assistenten ist in mehrere Registerkarten unterteilt, wobei jede Registerkarte klar Optionen zum Konfigurieren eines adaptiven Formulars enthält. Eine Autorin oder ein Autor von Formularen durchläuft die Registerkarten linear, um Optionen wie Vorlagen, Übermittlungsaktionen und Datenquellen für Formularkomponenten auszuwählen. <br/> ![Bild des Assistenten für die Formularerstellung](/help/release-notes/assets/wizard.png) <br/>Die Benutzeroberfläche des Assistenten optimiert den Prozess der Erkennung aller grundlegenden Optionen für ein adaptives Formular und erleichtert die Erstellung von Formularen, auch für Benutzende, die mit der Technologie nicht vertraut sind. |
| Adobe Analytics mit Automatisierung des Experience Cloud-Setups für Formulare | Die Formularanalyse bietet wertvolle Einblicke in die Leistung von Formularen, indem sie die Benutzerinteraktion misst, Konversionsraten optimiert, die Formularleistung überwacht und das Anwendererlebnis verbessert. Durch die Nachverfolgung des Benutzerverhaltens und des Feedbacks kann die Analyse Bereiche des Formulars identifizieren, die Frustration oder Verwirrung verursachen, und so zu Verbesserungen am Formularentwurf und an der Formularfunktionalität führen. <br/> <br/> Durch die Verwendung können Sie [Adobe Analytics mit der Automatisierung des Einrichtens von Experience Cloud](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/integrate/services/enable-adobe-analytics-adaptive-form-using-experience-cloud-setup-automation.html?lang=de) mit wenigen Klicks aktivieren. Dadurch können Sie AEM Forms as a Cloud Service mit Experience Platform-Tags verbinden und Adobe Analytics nutzen, um Leistungsmetriken für Ihre veröffentlichten Formulare zu erfassen und zu verfolgen. <br/> <br/> ![Adobe Analytics-Bericht zur Benutzerinteraktion im adaptiven Formular](/help/forms/assets/forms-analytics-report.png) <br/><br/>Forms as a Cloud Service bietet einen vorkonfigurierten Adobe Analytics-Bericht. Dies hilft Ihnen, die Leistung Ihrer Formulare einfach zu verstehen. Die Metriken auf Formularebene bieten Ihnen einen Einblick in die Leistung des Formulars in Bezug auf mehrere wichtige Leistungsindikatoren (KPIs) wie Ausgabedarstellungen, Besuchende, Übermittlungen, durchschnittliche Ausfüllzeit. <br/> <br/> Darüber hinaus enthält dies Informationen darüber, wie oft Benutzende im Durchschnitt auf die kontextbezogene Hilfe für die Felder im Bereich zugegriffen haben. So können Sie ermitteln, bei welchen Feldern Benutzende anhalten und erst Informationen einholen, bevor sie Informationen eingeben. Sie können solche Felder oder Hilfsinhalte weiter vereinfachen, um die Konversionen zu verbessern. |
